What is device finger print?

Device fingerprint identification (sometimes called PC fingerprint identification) is part of a broader technology called device intelligence to determine whether the computer you use should be trusted. Device fingerprinting is the measurement of anonymous browsers, operating systems, and connection properties to generate risk files for the device in real time. Device fingerprinting is a new way to distinguish valuable customers from professional fraudsters.

The prevalence of theft and hacking these days means it's very difficult to verify who you're dealing with. The new client could be a computer puppet representing an eastern European criminal gang, or a kleptomaniac who believes that facebook stole personal information. Obviously, online authentication is a huge challenge for all entrepreneurs.

Related technologies

The fingerprint of devices on the Internet has attracted extensive attention from research institutions and commercial markets. To date, almost all fingerprint features have been dependent on software running on the device, so that they can be changed by the user, thereby impeding access to the device's fingerprint. Based on this question, we believe that the nascent HTML5 standard has spawned a new class of hardware-based fingerprint features. Such features are harder to conceal or change, providing a higher degree of confidence for fingerprint identification. We propose several possible fingerprint methods that are implemented by allowing HTML5 Web applications to identify the device's hardware.

  • Cookie

    Created by NetScape, a small piece of data stored on a user's local terminal memory or disk can only hold string types, and all Cookie information is sent at the request of the browser.

  • UserData

    Introduced by Microsoft in IE5.0, the term refers to a piece of persistent string data stored at the user's local terminal. The data will be kept until the expiry time is manually deleted or set. Only ie5.0-ie9.0 is supported.

  • Flash Shared Object

    It allows you to store all flash supported data on your local client's hard drive or server. The data will be stored permanently, with no expiration time, and can be cleared by setting the manager or calling clear().

  • Google Gears

    An open source browser plugin released by Google in 2007 has an embedded SQL database based on SQLite and provides a unified API to access the database. After obtaining user authorization, each site can store unlimited data in the database.

  • WebStorge

    Webstorage is a new tag of HTML5, which is one of the solutions of localStorage, including sessionStorage and localStorage.

  • SimHash

    Simhash is a locally sensitive hash. So what is local sensitivity? Suppose you have a certain similarity between two strings, and you can still maintain that after the hash. It is used by Google to re-weight vast amounts of text.
